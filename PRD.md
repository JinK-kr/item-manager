# 공용 물품 관리 앱 PRD

| 항목 | 내용 |
|---|---|
| 제품명 | 우리 반 공용 물품 관리 앱 (가칭) |
| 문서 버전 | v1.0 |
| 작성일 | 2026-07-23 |
| 문서 목적 | 코드 작성 전에 만들 범위와 만들지 않을 범위를 확정한다 |

---

## 1. 배경과 목적

동아리방이나 실습실에는 여러 사람이 함께 쓰는 물품이 있다. 볼펜, A4 용지, 건전지, 물티슈 같은 것들이다.
지금은 이런 물품을 종이 목록이나 단체 채팅방에서 관리한다. 그래서 아래 문제가 생긴다.

- 누가 무엇을 얼마나 가져갔는지 기록이 남지 않는다.
- 재고가 떨어진 걸 필요한 순간에야 알게 된다.
- 물품이 실제로 몇 개 남았는지 아무도 정확히 모른다.

**이 앱의 목적은 하나다. 공용 물품을 여러 사람이 함께 등록하고, 수량을 바로바로 고쳐서, 지금 재고가 어떤지 한눈에 보게 만드는 것이다.**

성공했다고 볼 수 있는 기준은 다음과 같다.

1. 물품 하나를 등록하는 데 10초 안에 끝난다.
2. 물품을 쓴 사람이 그 자리에서 수량을 줄일 수 있다.
3. 대시보드만 보면 재고가 부족한 물품이 무엇인지 바로 알 수 있다.

## 2. 사용자

| 구분 | 설명 |
|---|---|
| 주 사용자 | 동아리원, 실습실 사용 학생 (10~30명 규모) |
| 사용 상황 | 물품을 쓰기 직전 또는 직후, 실습실에서 노트북이나 휴대폰으로 접속 |
| 전제 | 서로 아는 사이다. 그래서 로그인 없이 닉네임만 적는다. 잘못된 기록은 서로 확인해서 고친다. |

### 대표 사용 흐름

- **등록**: 부원이 새 물티슈 3개를 사 왔다 → index.html에서 이름(물티슈), 카테고리(청소용품), 수량(3), 닉네임을 넣고 등록한다.
- **사용**: 실습에 멀티탭 1개를 가져간다 → index.html 목록에서 멀티탭의 `−` 버튼을 누른다. 수정 시간이 자동으로 갱신된다.
- **확인**: 총무가 주 1회 dashboard.html을 연다 → 재고 부족 목록을 보고 살 물건을 정한다.

## 3. 범위

### 3.1 이번에 만드는 것

**화면은 두 개다.**

| 파일 | 역할 |
|---|---|
| `index.html` | 물품 등록 + 물품 목록 (수량 조절, 삭제 포함) |
| `dashboard.html` | 현황 대시보드 (위젯 4개) |

**기능은 딱 세 가지다.**

1. 물품 등록
2. 수량 늘리기와 줄이기
3. 물품 삭제
4. 엑셀 일괄 등록 *(나중에 추가됨 — 아래 참고)*

> **범위가 한 번 늘어났다.** 처음에는 위 세 가지만 만들기로 했다.
> 그런데 실습실 물품 수십 개를 폼으로 하나씩 넣는 게 이 앱에서 제일 번거로운 작업이었고,
> 그런 목록은 대개 이미 엑셀에 있었다. 그래서 네 번째 기능으로 엑셀 업로드를 넣었다.
> 대시보드 위젯은 그대로 네 개다.

**대시보드 위젯은 딱 네 개다.**

1. 총 품목 수
2. 재고 부족 목록 (수량 3개 이하)
3. 카테고리별 분포 도넛 차트
4. 최근 변경 내역

### 3.2 이번에 만들지 않는 것 (Out of Scope)

아래 기능은 **이번 버전에서 만들지 않는다.** 요청이 들어와도 다음 버전으로 미룬다.

| 만들지 않는 기능 | 이유 |
|---|---|
| **로그인 / 회원가입 / 계정 관리** | 서로 아는 사이라 필요 없다. 등록자 닉네임 입력으로 대신한다. |
| **검색 / 필터 / 정렬 UI** | 물품 수가 적어서 화면 하나에 다 보인다. 목록이 길어지면 그때 만든다. |
| **사진 업로드 / 이미지 첨부** | 저장 공간과 업로드 처리가 필요해서 범위가 커진다. (엑셀 파일 업로드는 별개다 — F-04 참고) |
| **대여와 반납 기능** | 누가 언제 빌려 갔는지 추적하는 별도 데이터 구조가 필요하다. 이번엔 수량 증감으로만 관리한다. |

추가로 이번 버전에서 다루지 않는 것: 물품 정보 수정(이름·카테고리 변경), 알림, 데이터 내보내기, 권한 구분, 삭제 취소(휴지통).

## 4. 데이터 모델

물품 하나는 아래 정보를 가진다.

| 필드 | 타입 | 필수 | 설명 | 입력 규칙 |
|---|---|---|---|---|
| `id` | 문자열 | O | 내부 식별자. 화면에는 보이지 않는다. | 등록 시 자동 생성 |
| `name` | 문자열 | O | 물품 이름 | 1~30자. 공백만 입력하면 안 된다. |
| `category` | 문자열 | O | 카테고리 | 아래 고정 목록에서 선택 |
| `quantity` | 정수 | O | 수량 | 0 이상의 정수. 음수가 될 수 없다. |
| `owner` | 문자열 | O | 등록자 닉네임 | 1~10자 |
| `updatedAt` | 날짜시간 | O | 수정 시간 | 자동 기록. 사용자가 직접 입력하지 않는다. |

### 카테고리 고정 목록 (4개)

카테고리는 아래 **네 가지로 고정**한다. 사용자가 직접 추가하거나 이름을 바꿀 수 없다.

| 카테고리 | 예시 물품 |
|---|---|
| **문구류** | 볼펜, 네임펜, A4 용지, 포스트잇, 가위, 테이프, 스테이플러 |
| **전자기기** | 건전지, 멀티탭, USB 케이블, 충전기, 계산기, 스피커 |
| **청소용품** | 물티슈, 쓰레기봉투, 걸레, 세제, 빗자루, 마른행주 |
| **기타** | 위 세 가지에 들어가지 않는 물품 (공구, 구급용품, 실습 재료 등) |

> 자유 입력이 아니라 드롭다운으로 고른다. 같은 물건을 "문구"와 "문구류"로 따로 적는 일을 막기 위해서다. 애매한 물품은 전부 `기타`로 넣는다. 이 네 가지가 그대로 도넛 차트의 항목이 된다.

### 수정 시간 규칙

- 물품을 **등록**할 때 현재 시각으로 기록한다.
- 수량을 **늘리거나 줄일** 때마다 현재 시각으로 갱신한다.
- 화면에는 `2026-07-23 18:40` 형식으로 보여 준다.

## 5. 기능 요구사항

### F-01. 물품 등록

- 위치: `index.html` 상단 등록 폼
- 입력 항목: 물품 이름(텍스트) · 카테고리(드롭다운) · 수량(숫자, 기본값 1) · 등록자 닉네임(텍스트)
- 동작
  1. `등록` 버튼을 누르면 입력값을 검사한다.
  2. 통과하면 목록에 새 물품을 추가하고 저장한다.
  3. 수정 시간을 현재 시각으로 기록한다.
  4. 폼을 비운다. 단, **등록자 닉네임은 지우지 않고 유지한다** (같은 사람이 연달아 여러 개를 등록하는 경우가 많다).
- 검증 규칙
  - 이름이 비었거나 공백뿐이면 "물품 이름을 입력해 주세요" 안내를 띄우고 등록하지 않는다.
  - 닉네임이 비었으면 "닉네임을 입력해 주세요" 안내를 띄운다.
  - 수량이 0 미만이거나 숫자가 아니면 등록하지 않는다.
  - **같은 이름의 물품이 이미 있어도 그대로 등록한다.** 중복 검사는 하지 않는다. (사용자가 판단할 몫이다.)

### F-02. 수량 늘리기와 줄이기

- 위치: `index.html` 목록의 각 행
- UI: 수량 숫자 양옆에 `−` 버튼과 `+` 버튼
- 동작
  - `+`를 누르면 수량이 1 늘어난다.
  - `−`를 누르면 수량이 1 줄어든다.
  - 변경할 때마다 수정 시간이 현재 시각으로 갱신된다.
  - 화면 새로고침 없이 그 자리에서 숫자가 바뀐다.
- 규칙
  - **수량은 0 아래로 내려가지 않는다.** 수량이 0이면 `−` 버튼을 비활성화한다.
  - 수량이 0이어도 물품은 목록에 남는다. 자동으로 삭제하지 않는다.
  - 수량이 3 이하가 되면 목록에서 해당 행을 눈에 띄게 표시한다 (예: 경고색 배지).

### F-03. 물품 삭제

- 위치: `index.html` 목록의 각 행 오른쪽 `삭제` 버튼
- 동작
  1. `삭제` 버튼을 누르면 "○○을(를) 삭제할까요?" 확인 창을 띄운다.
  2. 확인하면 목록에서 지우고 저장한다.
  3. 취소하면 아무 일도 일어나지 않는다.
- 규칙
  - **되돌리기(복구)는 없다.** 지우면 끝이다. 그래서 확인 창을 반드시 거친다.
  - 누구나 어떤 물품이든 지울 수 있다. 등록자만 지울 수 있게 막지 않는다 (로그인이 없으므로).

### F-04. 엑셀 일괄 등록

- 위치: `index.html` 의 `엑셀로 한 번에 등록하기` 칸. 왼쪽 메뉴의 `엑셀 업로드` 가 여기로 데려온다.
- 받는 파일: `.xlsx` · `.xls` · `.csv`. 첫 번째 시트만 읽는다.
- 첫 줄이 헤더다. `물품이름` · `카테고리` · `수량` · `등록자`
  - 공백과 대소문자는 무시하고 맞춘다 (`물품 이름` = `물품이름`)
  - `이름` `품명` `분류` `개수` `닉네임` 같은 다른 표기도 받아 준다
  - `등록자` 칸이 없거나 비면, 화면에 입력한 닉네임으로 채운다
- 동작
  1. 파일을 고르면 읽어서 줄마다 검사한다. **이 시점에는 아무것도 등록하지 않는다.**
  2. 무엇이 들어갈지 표로 보여 준다 — 신규 몇 건, 합산 몇 건, 건너뜀 몇 건.
  3. `등록하기`를 눌러야 실제로 저장한다. `취소`하면 아무 일도 없다.
- 규칙
  - **같은 이름이 이미 있으면 수량을 더한다.** 예) 볼펜 5개 + 엑셀 10개 → 볼펜 15개 한 줄
    - 이름 비교는 앞뒤 공백과 대소문자를 무시한다
    - 합산할 때 기존 카테고리와 등록자는 그대로 두고 수량만 바꾼다 (`+`/`−` 버튼과 같은 방식)
    - 같은 이름이 여러 줄이면 가장 최근에 손댄 줄에 더한다
  - **잘못된 줄만 건너뛴다.** 없는 카테고리, 숫자가 아닌 수량, 음수 등은 이유와 줄 번호를 보여 주고 넘어간다. 나머지 정상 줄은 그대로 등록된다.
  - 완전히 빈 줄은 조용히 무시한다.
  - 한 번에 **500줄까지**만 받는다.
  - 저장은 DB 함수 안에서 한 트랜잭션으로 한다. **중간에 실패하면 전부 취소된다** (절반만 등록되는 일이 없다).
- 왜 미리보기를 반드시 거치는가
  - 일괄 삭제 기능이 없다. 실수로 같은 파일을 두 번 올리면 수십 건을 하나씩 지워야 한다.
  - 확인 단계가 그걸 막는 유일한 장치다.
- 곁들이는 것: `양식 내려받기` — 헤더와 예시가 채워진 빈 엑셀을 만들어 준다.
  이 칸 안에도 있고 왼쪽 메뉴에도 있어서, 대시보드를 보다가도 바로 받을 수 있다.
- 엑셀 라이브러리(SheetJS, 약 900KB)는 **처음부터 받지 않는다.** 양식을 받거나 파일을 고를 때
  그때 받아 온다. 대부분은 `+`/`−` 만 누르고 가므로 그 사람들이 굳이 받을 이유가 없다.

## 6. 화면 요구사항

### 6.0 공통 뼈대 — 왼쪽 메뉴 + 오른쪽 내용

두 화면 모두 같은 왼쪽 메뉴를 쓴다. 메뉴 내용은 `sidebar.js` 한 곳에서 만든다.
두 HTML 에 같은 내용을 적어 두면 한쪽만 고치는 실수가 생기기 때문이다.

```
┌────────────────┬──────────────────────────────┐
│ 우리 반 물품 관리 │  (화면 제목)                  │
│                │                              │
│ 관리            │  ┌────────────────────────┐  │
│  ▪ 물품 목록 30 │  │                        │  │
│  ▪ 현황 대시보드 │  │   화면 내용             │  │
│                │  │                        │  │
│ 데이터          │  └────────────────────────┘  │
│  ▪ 엑셀 업로드   │                              │
│  ▪ 양식 내려받기 │                              │
└────────────────┴──────────────────────────────┘
```

| 메뉴 | 하는 일 |
|---|---|
| 물품 목록 | `index.html` 로 간다. 옆 배지에 지금 등록된 품목 수를 보여 준다 |
| 현황 대시보드 | `dashboard.html` 로 간다 |
| 엑셀 업로드 | `index.html#import` 로 가서 그 칸을 잠깐 밝혀 준다 |
| 양식 내려받기 | 화면 이동이 아니라 동작이다. 두 화면 어디서든 양식 파일을 만들어 준다 |

- 지금 보고 있는 화면의 메뉴는 강조해서 표시한다.
- **좁은 화면(900px 이하)에서는 메뉴를 감춘다.** 왼쪽 위 햄버거 버튼으로 열리고,
  가림막을 누르거나 `Esc` 를 누르거나 메뉴 항목을 고르면 닫힌다.
  실습실에서 폰으로 쓰는 경우가 많아서, 좁은 화면에서 메뉴가 본문을 잡아먹으면 안 된다.

### 6.1 index.html — 물품 등록과 목록

```
│  물품 목록                                │
├─────────────────────────────────────────┤
│  물품 등록                                │
│  [이름____] [카테고리▼] [수량_] [닉네임_] │
│                              [ 등록 ]     │
├─────────────────────────────────────────┤
│  엑셀로 한 번에 등록하기                   │
│  [양식 내려받기] [닉네임_] [파일 고르기]   │
├─────────────────────────────────────────┤
│  등록된 물품 (12개)                       │
│  ┌─────────────────────────────────────┐ │
│  │ 볼펜     문구류  [−] 5 [+] 민수  삭제│ │
│  │ 건전지   전자기기[−] 2 [+] 지연  삭제│ │  ← 부족 표시
│  │ ...                                  │ │
│  └─────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```

- 목록에 보여 줄 항목: 물품 이름 · 카테고리 · 수량(± 버튼 포함) · 등록자 닉네임 · 수정 시간 · 삭제 버튼
- 정렬: 수정 시간 최신순 (가장 최근에 손댄 물품이 맨 위)
- 물품이 하나도 없으면 "아직 등록된 물품이 없어요. 위에서 첫 물품을 등록해 보세요." 안내를 보여 준다.
- 화면 이동은 왼쪽 메뉴로 한다 (6.0 참고).

### 6.2 dashboard.html — 현황 대시보드

위젯은 **정확히 네 개**다. 더 늘리지 않는다.

| # | 위젯 | 보여 줄 내용 | 계산 방법 |
|---|---|---|---|
| 1 | **총 품목 수** | 큰 숫자 하나 | 등록된 물품 행의 개수. 수량 합계가 아니라 **품목 종류 수**다. (예: 볼펜 5개 + 건전지 2개 → 2) |
| 2 | **재고 부족 목록** | 물품 이름 + 현재 수량 리스트 | 수량이 **3 이하**인 물품 전체. 수량 오름차순 정렬(적은 것부터). 없으면 "부족한 물품이 없어요" 표시 |
| 3 | **카테고리별 분포** | 도넛 차트 + 범례 | 카테고리별 **품목 수** 기준. 물품이 0개인 카테고리는 차트에 넣지 않는다. |
| 4 | **최근 변경 내역** | 물품 이름 / 수량 / 닉네임 / 수정 시간 | 수정 시간 최신순 **상위 5건** |

- 화면 이동은 왼쪽 메뉴로 한다 (6.0 참고).
- 등록된 물품이 하나도 없으면 위젯마다 빈 상태 문구를 보여 준다. 오류를 내거나 빈 화면을 보여 주면 안 된다.

> **최근 변경 내역 구현 참고**: 물품마다 저장하는 정보는 `수정 시간` 하나다. 그래서 이 위젯은 별도 이력 테이블 없이, 물품 목록을 수정 시간 내림차순으로 정렬해 위에서 5개를 보여 주는 방식으로 만든다. "수량이 5에서 4로 줄었다" 같은 변화량은 이번 버전에서 보여 주지 않는다.

## 8. 가정과 미결 사항

### 저장소 변경 이력

처음 버전은 데이터를 브라우저 `localStorage`에 저장했다. 그래서 기기 한 대에서 번갈아 쓰는 구조였고, 내 휴대폰에서 등록한 물품이 옆 사람 휴대폰에 나타나지 않았다.

**지금은 Supabase(PostgreSQL)로 옮겼다.** 각자 기기에서 같은 목록을 본다. 이 문서 1절의 "여러 사람이 함께 등록하고 관리한다"는 목적이 이제 제대로 지켜진다.

### 짚고 넘어갈 점: 아무나 지울 수 있다

로그인을 만들지 않기로 했으므로, 테이블 접근 정책(RLS)을 **누구나 읽기·추가·수정·삭제** 로 열어 두었다.

프로젝트 주소와 `anon` 키는 브라우저로 전송되는 값이라 숨길 수 없다. 저장소가 공개라면 **인터넷의 누구나 이 표를 읽고 고치고 전부 지울 수 있다.** 반 안에서 쓰는 물품 목록이라 감수하고 택한 설정이다.

조여야 할 때 쓸 수 있는 방법은 세 가지다. 저장소를 비공개로 돌리거나, 삭제 정책만 막거나, Supabase 익명 로그인을 켜고 정책을 `authenticated`로 좁힌다. **이번 범위에서는 하지 않는다.**

### 아직 안 되는 것: 실시간 반영

다른 사람이 바꾼 내용은 **화면을 새로 고쳐야 보인다.** Supabase Realtime을 붙이면 즉시 반영되지만, 이번 범위에서는 하지 않는다.

### 그 외 결정 사항

| 항목 | 이번 버전의 결정 |
|---|---|
| 같은 이름 물품 중복 등록 | 막지 않는다 |
| 수량 0인 물품 | 목록에 그대로 둔다. 자동 삭제하지 않는다 |
| 삭제 복구 | 없다. 확인 창으로만 방지한다 |
| 카테고리 추가/편집 | 없다. 코드에 고정된 4개(문구류·전자기기·청소용품·기타)만 쓴다 |
| 재고 부족 기준값 | 3개 이하로 고정한다. 설정 화면은 만들지 않는다 |

## 9. 완료 기준 (체크리스트)

아래가 모두 되면 이번 버전은 끝이다.

**index.html**
- [ ] 이름, 카테고리, 수량, 닉네임을 넣고 등록하면 목록에 나타난다
- [ ] 이름이나 닉네임이 비면 안내가 뜨고 등록되지 않는다
- [ ] `+`를 누르면 수량이 늘고 수정 시간이 갱신된다
- [ ] `−`를 누르면 수량이 줄고 수정 시간이 갱신된다
- [ ] 수량이 0일 때 `−`를 눌러도 음수가 되지 않는다
- [ ] 수량 3개 이하인 행에 부족 표시가 보인다
- [ ] 삭제 버튼을 누르면 확인 창이 뜨고, 확인하면 사라진다

**엑셀 일괄 등록 (F-04)**
- [ ] `양식 내려받기`로 헤더가 채워진 엑셀이 받아진다
- [ ] 파일을 고르면 미리보기 표가 뜨고, 이 시점에는 아직 등록되지 않는다
- [ ] 신규 / 합산이 구분돼 보이고, 합산은 `5 + 10 = 15` 처럼 결과가 보인다
- [ ] 없는 카테고리·잘못된 수량인 줄은 이유와 줄 번호가 표시되고 건너뛴다
- [ ] `등록하기`를 눌러야 저장되고, `취소`하면 아무 일도 없다
- [ ] 같은 파일을 두 번 올리면 줄이 늘지 않고 수량만 더해진다
- [ ] 새로고침해도 데이터가 남아 있다
- [ ] 물품이 없을 때 빈 상태 안내가 보인다

**dashboard.html**
- [ ] 총 품목 수가 실제 목록 개수와 같다
- [ ] 재고 부족 목록에 수량 3 이하 물품만 나온다
- [ ] 도넛 차트가 카테고리별 품목 수를 보여 준다
- [ ] 최근 변경 내역에 수정 시간 최신순 5건이 나온다
- [ ] 물품이 하나도 없어도 오류 없이 빈 상태로 보인다

**왼쪽 메뉴**
- [ ] 두 화면 모두 같은 메뉴가 보이고, 지금 보는 화면이 강조된다
- [ ] `물품 목록` 옆 배지에 실제 품목 수가 나온다
- [ ] `엑셀 업로드`를 누르면 업로드 칸으로 가서 잠깐 밝혀진다

- [ ] 햄버거로 열리고, 가림막·`Esc`·메뉴 선택으로 닫힌다
