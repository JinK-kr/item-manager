# (주)길동물산 물품 관리

사무실·창고의 공용 물품을 여러 사람이 함께 등록하고 수량을 관리하는 앱입니다.
데이터는 **Supabase(PostgreSQL)** 에 저장돼서, 여러 사람이 각자 기기에서 같은 목록을 봅니다.

## 화면

왼쪽에 메뉴가 있고 오른쪽에 내용이 나옵니다.
좁은 화면에서는 메뉴가 숨고, 왼쪽 위 햄버거 버튼으로 열립니다.

| 메뉴 | 하는 일 |
|---|---|
| **현황 대시보드** | `index.html` — 위젯 4개. **주소만 치면 여기가 먼저 나옵니다** |
| **물품 목록** | `items.html` — 검색·정렬, 수량 조절, 삭제. 옆 배지는 현재 품목 수 |
| **물품 등록** | `register.html` — 물품 하나 등록 |
| **엑셀 업로드** | `import.html` — 양식 받기 · 파일 올리기 · 미리보기 |
| **양식 내려받기** | 어느 화면에서든 빈 양식 엑셀을 받습니다 |

메뉴 아래쪽에서 **화면 밝기**를 고를 수 있습니다.
`시스템`(기본, 컴퓨터 설정을 따름) · `밝게` · `어둡게` 세 가지이고, 고른 값은 다음에 열 때도 유지됩니다.

## 기능

- **물품 등록** — 이름, 카테고리, 수량, **적정 재고량**, **입고일**, 등록자 닉네임
- **검색과 정렬** — 이름·카테고리·등록자로 찾고, 최신순·이름순·수량순으로 정렬 (다시 누르면 방향 반대)
- **수량 늘리기 / 줄이기** — `+` `−` 버튼. 0 아래로는 내려가지 않습니다
- **물품 삭제** — 확인 창을 거칩니다 (되돌릴 수 없음)
- **엑셀 일괄 등록** — 파일 하나로 수십 건을 한 번에 (아래 설명)
- **말로 등록** — `볼펜 20개랑 물티슈 5개 들어왔어` 라고 적으면 알아서 나눠 줍니다 (아래 설명)

카테고리는 `문구류` · `전자기기` · `청소용품` · `기타` 네 가지로 고정입니다.

## 말로 등록하기

`물품 등록` 화면 위쪽에 있습니다. 문장으로 적으면 나눠서 정리해 줍니다.

> **입력** `회의실에 멀티탭 3개랑 HDMI 케이블 2개 갖다놨어`
>
> | 상태 | 이름 | 카테고리 | 수량 |
> |---|---|---|---|
> | 신규 | HDMI 케이블 | 전자기기 ▼ | 2 |
> | 합산 | 멀티탭 | 전자기기 ▼ | 4 + 3 = 7 |

- **바로 저장하지 않습니다.** 무엇이 들어갈지 먼저 보여 주고, 확인하시면 그때 저장합니다.
- **날짜도 알아듣습니다.** `어제 볼펜 20개 들어왔어` → 입고일이 어제로 잡힙니다.
- **쓸 모델을 고를 수 있습니다.** 목록에 **무료 모델만** 뜹니다 (아래 설명).
- **카테고리는 고칠 수 있습니다.** 이름·수량은 말한 그대로지만 카테고리는 모델이 추측한 값이라, 틀렸으면 드롭다운으로 바꾸세요.
- **등록만 됩니다.** 지우기·수량 줄이기는 `물품 목록`에서 하세요. 애매한 문장으로 되돌릴 수 없는 일이 일어나면 안 되니까요.
- 안 되면 바로 아래 **직접 입력** 폼을 쓰시면 됩니다.

### 무료 모델만 씁니다

고를 수 있는 모델은 셋뿐이고 전부 무료입니다.

| 모델 | 형식 강제 |
|---|---|
| `google/gemma-4-26b-a4b-it:free` | JSON 스키마 (가장 안정적) |
| `google/gemma-4-31b-it:free` | json_object |
| `nvidia/nemotron-3-nano-omni-30b-a3b-reasoning:free` | 없음 (프롬프트로만) |

돈이 붙는 모델은 **세 겹으로 막습니다.**

1. 위 목록에 있는 id 여야 함
2. id 가 `:free` 로 끝나야 함
3. **호출 직전에 오픈라우터에서 실제 가격을 받아 전부 0원인지 확인**

하나라도 어긋나면 부르지 않고 거절합니다. 가격을 코드에 적어 두지 않고 매번 확인하므로,
나중에 어떤 모델이 유료로 바뀌어도 자동으로 걸러집니다.

> 한 겹 더 두시려면 오픈라우터 → Keys 에서 그 키의 **크레딧 한도를 0** 으로 걸어 두세요.

### API 키는 어디에 있나

오픈라우터 키는 **브라우저에도 저장소에도 없습니다.** `.env` 파일도 쓰지 않습니다.

```
브라우저 ──▶ parse-items (Supabase 함수) ──키 붙여서──▶ 오픈라우터
                     ▲
        Supabase secrets 의 OPENROUTER_API_KEY
```

브라우저 코드는 누구나 F12로 볼 수 있어서, 키를 넣으면 그대로 노출됩니다.
`.env` 도 답이 아닙니다 — 이 앱은 서버가 없어 읽을 주체가 없고, 저장소에 올리면 공개됩니다.
그래서 Supabase 함수를 중간에 두고 키를 거기에 보관합니다.

함수 코드는 [`supabase/functions/parse-items/index.ts`](supabase/functions/parse-items/index.ts) 에 있습니다.
**키 값은 코드 어디에도 없습니다.** `Deno.env.get('OPENROUTER_API_KEY')` 로 이름만 부릅니다.

## 엑셀로 한 번에 등록하기

처음 재고를 넣을 때 폼으로 하나씩 치는 게 제일 번거롭습니다.
왼쪽 메뉴의 **엑셀 업로드**를 누르면 전용 화면이 열립니다. 세 단계로 안내합니다.

1. **양식 받기** — 헤더와 예시가 채워진 빈 엑셀을 받습니다
2. **채우기** — 둘째 줄부터 물품을 적습니다
3. **올리기** — 파일을 끌어다 놓거나 눌러서 고릅니다.
   **무엇이 등록될지 표로 먼저 보여 주고**, 확인하시면 그때 저장합니다

| 항목 | 내용 |
|---|---|
| 파일 형식 | `.xlsx` · `.xls` · `.csv` (첫 번째 시트만 읽습니다) |
| 헤더 | `물품이름` `카테고리` `수량` `등록자` `입고일` `적정재고량` — 공백·대소문자는 무시하고 맞춥니다 |
| 등록자가 빈 줄 | 화면에 적은 닉네임으로 채웁니다 |
| 입고일·적정재고량이 빈 줄 | 각각 오늘 날짜와 5개로 채웁니다 |
| **같은 이름이 있으면** | **수량을 더합니다.** 볼펜 5개 + 엑셀 10개 → 볼펜 15개 (줄이 늘지 않습니다) |
| 잘못된 줄 | 그 줄만 이유와 함께 건너뛰고, 나머지는 정상 등록됩니다 |
| 한도 | 한 번에 500줄 |

읽자마자 등록하지 않고 미리보기를 거치는 이유가 있습니다. **일괄 삭제 기능이 없어서**,
실수로 같은 파일을 두 번 올리면 되돌리기가 번거롭습니다. 확인 단계가 그걸 막습니다.

## 대시보드 위젯

1. **총 품목 수** — 등록된 물품 종류 수
2. **재고 부족** — 적정 재고량에 못 미치는 물품. 급한 5건만 보이고 나머지는 목록으로 연결
3. **카테고리별 분포** — 품목 수 기준 도넛 차트
4. **최근 변경 내역** — 수정 시간 최신순 5건

## 처음 설정하기

### 1. 데이터베이스 만들기

Supabase 대시보드 → 프로젝트 선택 → 왼쪽 메뉴 **SQL Editor** → **New query** 를 누르고
[`supabase/schema.sql`](supabase/schema.sql) 전체를 붙여넣은 뒤 **Run** 을 누릅니다.

이 SQL 이 만드는 것:

| 항목 | 내용 |
|---|---|
| `items` 표 | 이름·카테고리·수량·등록자 닉네임·수정 시간 |
| 트리거 | 값이 바뀌면 수정 시간을 자동으로 갱신 |
| `change_quantity()` 함수 | 수량 증감을 DB 안에서 한 번에 처리 (아래 설명) |
| `import_items()` 함수 | 엑셀 일괄 등록. 같은 이름이면 수량을 더함 |
| RLS 정책 4개 | 누구나 읽기·추가·수정·삭제 가능 |

여러 번 실행해도 안전하게 짜여 있습니다.

### 2. 주소와 키 넣기

Supabase 대시보드 → **Project Settings** → **API** 에서 두 값을 복사해
[`supabase-config.js`](supabase-config.js) 에 넣습니다.

| 복사할 값 | 넣을 곳 |
|---|---|
| Project URL | `url` |
| Publishable key (`sb_publishable_...`) | `publicKey` |

`service_role` 키는 절대 넣으면 안 됩니다. 그 키는 모든 정책을 무시합니다.

### 3. 열기

`index.html` 을 더블클릭하면 열립니다. 브라우저가 로컬 파일에서 막는 경우에는 이 폴더에서 서버를 띄우세요.

```bash
python -m http.server 8765
```

그다음 `http://localhost:8765/` 로 접속합니다.

설정이 덜 끝났으면 화면 위에 무엇이 빠졌는지 안내가 뜹니다.

## 수량 증감을 DB 함수로 처리하는 이유

앱에서 "현재 값을 읽고 → 1 더하고 → 다시 쓰기" 를 하면, 두 사람이 거의 동시에 `+` 를 눌렀을 때
둘 다 5 를 읽고 둘 다 6 을 써서 7 이 되지 않습니다. 한 번이 사라집니다.

그래서 `change_quantity(item_id, delta)` 라는 DB 함수를 만들어 한 번에 처리합니다.
0 아래로 내려가지 않게 막는 것도 이 함수 안에서 합니다.

## 코드를 고쳐서 다시 올릴 때

HTML 안의 파일 주소에 `?v=5` 같은 번호가 붙어 있습니다.

```html
<link rel="stylesheet" href="style.css?v=5">
<script src="app.js?v=5"></script>
```

**CSS 나 JS 를 고쳤으면 이 번호를 올려 주세요** (`?v=5` → `?v=6`).
`index.html` · `items.html` · `register.html` · `import.html` **네 개 모두** 바꿔야 합니다.

번호를 안 올리면, 이미 접속했던 사람은 브라우저에 남아 있는 예전 파일을 계속 씁니다.
새 HTML 에 옛 CSS 가 물려서 화면이 깨져 보일 수 있습니다.
GitHub Pages 는 파일을 10분간 캐시하라고 알려 주기 때문에, 배포 직후에 특히 잘 생깁니다.

## 파일 구성

```
index.html / dashboard.js      현황 대시보드 (첫 화면)
items.html / items.js          물품 목록 (검색·정렬)
register.html / register.js    물품 등록
import.html / import.js        엑셀 업로드 화면
supabase/functions/parse-items/ 말을 물품으로 바꾸는 Edge Function
sidebar.js                     왼쪽 메뉴 + 화면 밝기 (네 화면이 함께 씀)
app.js                         네 화면이 공유하는 데이터 계층 (Supabase)
supabase-config.js             내 프로젝트 주소와 publishable 키
supabase/schema.sql            표·트리거·함수·정책을 만드는 SQL
style.css                      공용 스타일 (밝게/어둡게)
PRD.md                         제품 요구사항 문서
```

## 보안에 대해 알아 둘 점

이 앱은 로그인이 없습니다. 그래서 RLS 정책을 **누구나 읽고 쓸 수 있게** 열어 두었습니다.

`supabase-config.js` 에 적힌 주소와 publishable 키는 브라우저로 전송되므로 숨길 수 없습니다.
저장소가 공개라면 **인터넷의 누구나 이 표를 읽고, 고치고, 전부 지울 수 있습니다.**
사내에서 쓰는 물품 목록이라 감수하고 택한 설정입니다.

조여야 할 때 쓸 수 있는 방법:

- 저장소를 비공개로 돌린다
- 삭제 정책만 막고 앱에서 삭제 버튼을 숨긴다
- Supabase 익명 로그인을 켜고 정책을 `authenticated` 로 좁힌다

## 이번 버전에서 만들지 않은 기능

로그인 · 사진 업로드 · 대여와 반납 기능은 범위에서 제외했습니다.
대화로는 등록만 되고, 지우기·수량 줄이기는 안 됩니다 (일부러 막았습니다).
카테고리 필터, 물품 정보 수정(이름·카테고리 변경), 일괄 삭제도 아직 없습니다.
다른 사람이 바꾼 내용은 화면을 새로 고쳐야 보입니다 (실시간 반영은 다음 과제).
자세한 배경과 결정 사항은 [PRD.md](PRD.md) 에 정리돼 있습니다.
