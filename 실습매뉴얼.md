# AI로 물품 관리 웹앱 만들기 — 실습 매뉴얼

클로드 코드에게 말로 시켜서 **실제로 돌아가는 웹앱**을 만듭니다.
데이터베이스에 저장되고, 인터넷 주소로 남에게 보여줄 수 있고, AI로 물품을 등록합니다.

**완성 예시** — https://jink-kr.github.io/item-manager/

---

## 무엇을 만드나

| 화면 | 하는 일 |
|---|---|
| 현황 대시보드 | 총 품목 수, 재고 부족, 카테고리 분포, 최근 변경 |
| 물품 목록 | 검색·정렬, 수량 조절, 삭제 |
| 물품 등록 | 폼으로 하나씩 + **말로 등록** (AI) |
| 엑셀 업로드 | 파일 하나로 수십 건 한 번에 |

---

## 시작 전에

### 준비물

- **클로드 코드** (설치되어 있어야 합니다)
- 구글 계정 (깃허브·Supabase 가입에 씁니다)
- 크롬 같은 최신 브라우저

### 돈이 드나요

**전부 무료입니다.** 카드 등록도 필요 없습니다.

| 서비스 | 쓰는 것 | 비용 |
|---|---|---|
| GitHub | 저장소 + Pages(웹사이트 공개) | 무료 |
| Supabase | 데이터베이스 + 서버 함수 | 무료 플랜 |
| OpenRouter | AI 모델 | `:free` 모델만 사용 |

### 걸리는 시간

준비 30분 + 만들기 2~3시간. 한 번에 다 안 해도 됩니다.

### 작업 폴더

빈 폴더를 하나 만들고 **클로드 코드를 그 폴더에서 엽니다.**
폴더 이름에 한글이 있어도 됩니다. (예: `물품관리앱`)

---

# 준비 1. OpenRouter API 키 발급

AI 모델을 부르는 데 쓰는 열쇠입니다. **9단계에서만 필요**하니 나중에 해도 됩니다.

### 1-1. 가입하고 키 만들기

1. https://openrouter.ai 접속 → 구글 계정으로 로그인
2. 오른쪽 위 프로필 → **Keys** → **Create Key**
3. 이름은 아무거나 (예: `수업용`)
4. **Credit limit 을 0 으로** 걸어 두세요 ← 중요
5. `sk-or-v1-...` 로 시작하는 값이 나옵니다

> ⚠️ **이 창을 닫으면 다시 못 봅니다.** 바로 준비 3의 Supabase 화면에 붙여넣으세요.
> 놓쳤으면 옛 키를 지우고 새로 만들면 됩니다.

### 1-2. 무료 모델 이름 확인

무료 모델은 **자주 바뀌고 사라집니다.** 실습 당일에 직접 확인하세요.

https://openrouter.ai/models 에서 **FREE** 로 거르면 목록이 나옵니다.
이름 끝이 `:free` 인 것만 무료입니다.

**두세 개를 골라 적어 두세요.** 하나가 막혔을 때 갈아탈 수 있게요.

```
예) 2026년 7월 기준
  google/gemma-4-26b-a4b-it:free
  google/gemma-4-31b-it:free
  nvidia/nemotron-3-nano-omni-30b-a3b-reasoning:free
```

> **고르는 요령**: 모델 상세 페이지에서 `structured_outputs` 를 지원하는 걸 우선 고르세요.
> AI가 정해진 형식으로 답하게 강제할 수 있어서 훨씬 안정적입니다.

   - ~~Add a README file~~

처음 푸시할 때 깃허브 로그인 창이 뜹니다. 로그인하면 다음부터는 자동입니다.

# 준비 3. Supabase 설정하기

데이터를 저장하는 곳입니다. 물품 목록이 여기에 쌓입니다.

### 3-1. 프로젝트 만들기

1. https://supabase.com 가입 (깃허브 계정으로 가능)
2. **New project**
3. 이름은 아무거나, 리전은 **Northeast Asia (Seoul)** 또는 가까운 곳
4. 데이터베이스 비밀번호는 적어 두세요 (실습에선 안 쓰지만 분실하면 곤란합니다)
5. 만들어지는 데 1~2분 걸립니다

### 3-2. AI 키를 Supabase에 보관 (준비 1을 했다면)

1. 왼쪽 메뉴 **Edge Functions** → **Secrets**
2. 두 칸을 채우고 **Save**

| 칸 | 값 |
|---|---|
| Name | `OPENROUTER_API_KEY` ← **철자 그대로** |
| Value | `sk-or-v1-...` (준비 1에서 받은 값) |

3. 아래 **Custom secrets** 목록에 이름이 뜨면 성공입니다. 값은 안 보이는 게 정상입니다.

> ### 🔑 왜 `.env` 파일을 안 쓰나요
>
> 학생들이 가장 많이 하는 질문입니다.
>
> `.env` 는 **서버가 읽는 파일**입니다. 그런데 우리가 만드는 앱은 서버가 없습니다 —
> 브라우저에서만 돕니다. 그래서 `.env` 를 만들어도 **읽을 주체가 없습니다.**
>
> - `.env` 를 깃허브에 안 올리면 → 앱이 키를 못 찾아서 **동작 안 함**
> - 올리면 → **키가 그대로 공개됨** (봇이 몇 분 안에 긁어갑니다)
> - JS 파일에 직접 쓰면 → **F12 누르면 다 보임**
>
> 그래서 **Supabase 서버에 키를 맡깁니다.** 브라우저는 Supabase 함수에게만 말을 걸고,
> 그 함수가 대신 AI를 부릅니다. 키는 브라우저에도 깃허브에도 없습니다.
>
> ```
> 브라우저 ──▶ Supabase 함수 ──키 붙여서──▶ OpenRouter
>                    ▲
>            여기에만 키가 있음
> ```

### 3-3. 커넥터 연결 ★ 중요

클로드 코드가 Supabase를 직접 만질 수 있게 연결합니다.

1. 클로드 코드에서 **디렉터리 → 커넥터** 를 엽니다
2. **Supabase** 를 찾아 설치하고 연결(로그인)합니다
3. **⚠️ 클로드 코드를 껐다가 다시 켜세요**

> 커넥터는 **시작할 때 한 번** 연결됩니다. 켜져 있는 상태에서 추가하면
> 그 대화에서는 안 잡힙니다. "이미 설치되어 있습니다" 라고 나오는데
> 정작 클로드는 못 쓰는 상황이 이래서 생깁니다.

**연결 확인** — 새 대화에서 이렇게 물어보세요.

```
Supabase 커넥터가 연결됐는지 확인하고, 내 프로젝트 목록을 보여줘.
```

프로젝트 이름이 나오면 성공입니다.

# 1단계. PRD 만들기

**PRD** = 무엇을 만들지 미리 적어 두는 문서입니다.

바로 코드부터 만들면 AI가 이것저것 붙여서 산으로 갑니다.
**"무엇을 만들지"와 "무엇을 안 만들지"를 먼저 못 박아 두는 게 핵심입니다.**

### 프롬프트

```
사무실 공용 물품 관리 앱을 만들려고 해.

주요 기능은 물품 등록, 수량 늘리기와 줄이기, 물품 삭제
이 세 가지야.

사무실에서 공용 물품(볼펜, A4용지, 건전지, 물티슈 등)을
함께 쓰는 직원 10~30명을 위한 것이고,

누가 무엇을 얼마나 가져갔는지 기록이 안 남아서
재고가 떨어진 걸 필요한 순간에야 알게 되는 문제를 해결하기
위해 필요해.

어떻게 구현하면 좋을지
기술적인 방향과 함께
구체적인 PRD를 작성해 줘. PRD.md 파일로 만들어 줘.

아래 내용을 꼭 포함해 줘.
- 화면은 네 개다: 대시보드(첫 화면), 물품 목록, 물품 등록, 엑셀 업로드
- 물품 정보: 물품 이름, 카테고리, 수량, 적정 재고량, 입고일, 등록자 닉네임, 수정 시간
- 카테고리는 문구류 / 전자기기 / 청소용품 / 기타 네 가지로 고정
- 대시보드 위젯은 딱 네 개: 총 품목 수, 재고 부족, 카테고리별 분포 도넛 차트, 최근 변경 내역
- 데이터는 Supabase에 저장한다
- 이번에 만들지 않을 기능도 명확히 적어 줘:
  로그인, 사진 업로드, 대여와 반납 기능은 만들지 않는다.
```

### 확인할 것

- `PRD.md` 파일이 생겼나
- **"만들지 않는 기능"** 항목이 들어갔나 ← 이게 있어야 나중에 안 흔들립니다
- 카테고리가 네 개로 고정됐나

> **팁**: PRD를 읽어 보고 마음에 안 드는 부분은 그냥 말로 고치면 됩니다.
> `"카테고리에 예시 물품을 추가해 줘"` 처럼요.

# 2단계. 데이터베이스 만들기

### 프롬프트

```
PRD대로 Supabase에 items 테이블을 만들어 줘.
로그인이 없으니 누구나 읽고 쓸 수 있게 접근 정책도 함께 설정해 줘.

Supabase 대시보드 → **Table Editor** 에 `items` 테이블이 보이면 성공입니다.

컬럼이 이렇게 있어야 합니다.

```
id · name · category · quantity · target_quantity · received_at · owner · updated_at
```

> **여기서 배우는 것**: 클로드가 SQL을 직접 실행합니다.
> Supabase 화면에서 손으로 표를 만들 필요가 없습니다.

# 3단계. 화면 만들기

### 프롬프트

```
PRD대로 앱을 만들어 줘.

- 왼쪽에 메뉴를 두고 오른쪽에 내용이 나오는 구조로 해 줘.
  좁은 화면에서는 메뉴가 숨고 햄버거 버튼으로 열리게 해 줘.
- 메뉴는 현황 대시보드 / 물품 목록 / 물품 등록 이렇게 세 개.
- 첫 화면(index.html)은 대시보드로 해 줘.
- 물품 목록 화면에는 검색창과 정렬 버튼(최신순·이름순·수량순)을 넣어 줘.
- 화면 밝기를 시스템/밝게/어둡게 중에 고를 수 있게 해 줘.

다 만들면 브라우저에서 실제로 눌러보고 잘 되는지 확인해 줘.
```

### 확인할 것

클로드가 알아서 브라우저를 띄워 확인합니다. 끝나면 직접 눌러 보세요.

- [ ] 물품을 등록하면 목록에 나타나나
- [ ] `+` `−` 로 수량이 바뀌나
- [ ] 수량 0에서 `−` 가 안 눌리나
- [ ] 삭제할 때 확인 창이 뜨나
- [ ] 대시보드 위젯 네 개가 다 보이나
- [ ] 브라우저 창을 좁히면 메뉴가 숨나

> **안 되는 게 있으면** 그냥 말하면 됩니다.
> `"수량 0인데 마이너스 버튼이 눌려. 고쳐 줘"`

# 4단계. 깃허브에 올리고 웹사이트로 공개하기

### 4-1. 올리기

```
지금까지 만든 걸 깃허브에 커밋하고 푸시해 줘.
```

### 4-2. 웹사이트로 공개하기

1. `https://github.com/[내아이디]/item-manager/settings/pages` 접속
2. **Source** → `Deploy from a branch`
3. **Branch** → `main` / `/ (root)` → **Save**
4. 1~2분 뒤 주소가 생깁니다

```
https://[내아이디].github.io/item-manager/
```

### 4-3. 캐시 문제 미리 막기 ★

```
파일 주소에 ?v=1 같은 버전 번호를 붙여서
코드를 고쳐도 브라우저가 예전 파일을 쓰지 않게 해 줘.
```

> **왜 필요한가**: 이걸 안 하면 코드를 고쳐 올려도 **화면이 깨져 보입니다.**
> 브라우저가 새 HTML과 옛 CSS를 섞어 쓰기 때문입니다.
> 실습 중 "분명 고쳤는데 그대로예요" 의 90%가 이 문제입니다.
>
> 그래도 이상하면 **`Ctrl` + `Shift` + `R`** (강제 새로고침)을 누르세요.

# 5단계. 엑셀 업로드 기능

물품 40개를 폼으로 하나씩 넣는 건 고문입니다. 파일로 한 번에 넣습니다.

### 프롬프트

```
엑셀 파일을 올리면 물품이 한 번에 등록되는 기능을 만들어 줘.
왼쪽 메뉴에 '엑셀 업로드' 를 넣고 전용 화면으로 만들어 줘.

- 헤더가 채워진 빈 양식을 내려받는 버튼도 넣어 줘.
- 파일을 끌어다 놓아도 되게 해 줘.
- 읽자마자 등록하지 말고, 무엇이 들어갈지 표로 먼저 보여 준 다음
  확인을 눌러야 저장되게 해 줘.
- 이미 있는 물품과 이름이 같으면 수량을 더해 줘.
- 잘못된 줄은 그 줄만 건너뛰고 이유를 알려 줘.
```

### 확인할 것

- [ ] 양식을 받아서 몇 줄 채운 뒤 올려 보기
- [ ] **미리보기가 먼저 뜨나** (바로 등록되면 안 됩니다)
- [ ] 같은 파일을 두 번 올렸을 때 줄이 늘지 않고 수량만 더해지나

> **왜 미리보기가 중요한가**: 일괄 삭제 기능이 없어서, 실수로 두 번 올리면
> 수십 건을 하나씩 지워야 합니다. 확인 단계가 그걸 막는 유일한 장치입니다.

# 6단계. 말로 등록하기 (AI 연동) ★

이 실습의 하이라이트입니다.
`볼펜 20개랑 물티슈 5개 들어왔어` 라고 적으면 알아서 두 건으로 나눠 줍니다.

**준비 1(API 키)과 준비 3-2(Supabase에 키 저장)를 먼저 끝내야 합니다.**

### 프롬프트

```
물품 등록 화면에서 말로 등록할 수 있게 만들어 줘.
"볼펜 20개랑 물티슈 5개 들어왔어" 처럼 적으면 두 건으로 나눠 주는 거야.

- OpenRouter를 쓰는데, API 키는 브라우저에 절대 넣지 마.
  Supabase Edge Function을 만들어서 거기서 부르고,
  키는 Supabase 시크릿의 OPENROUTER_API_KEY를 읽게 해 줘.

- 무료 모델만 써야 해. 아래 세 개만 고를 수 있게 해 줘.
  [준비 1-2에서 적어 둔 모델 이름 세 개를 여기에]
  호출하기 전에 실제로 무료가 맞는지 확인하고, 유료면 거절하게 해 줘.

- 화면에서 모델을 고를 수 있게 해 줘.

- 바로 저장하지 말고 미리보기를 보여 준 다음 확인을 받아 줘.

# 자주 막히는 곳

| 증상 | 원인과 해결 |
|---|---|
| 화면이 스타일 없이 깨져 보임 | 브라우저 캐시. `Ctrl`+`Shift`+`R` |
| 커넥터를 설치했는데 클로드가 못 씀 | **클로드 코드 재시작** 필요 |
| "OPENROUTER_API_KEY를 찾지 못했어요" | Supabase 시크릿 이름 오타 확인 |
| AI가 엉뚱한 답을 함 | 다른 무료 모델로 바꿔 보기 |
| 무료 모델이 사라짐 | https://openrouter.ai/models 에서 새로 고르기 |
| Pages 주소가 404 | Settings → Pages 에서 켰는지 확인, 2분 대기 |
| 고쳤는데 안 바뀜 | 버전 번호(`?v=`)를 올렸는지 확인 |

# ⚠️ 수업이 끝나면

이 앱은 **로그인이 없어서 주소를 아는 사람은 누구나 고치고 지울 수 있습니다.**
공개 저장소라 접속 정보도 공개되어 있습니다. 연습용이라 감수한 설정입니다.

발표가 끝나면 정리하세요.

1. Supabase → Edge Functions → 함수 삭제
2. Supabase → Secrets → `OPENROUTER_API_KEY` 삭제
3. OpenRouter → Keys → 키 폐기
4. 계속 두려면 저장소를 **Private** 으로 전환
